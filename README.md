# 星际公民描述文本自动填充器

使用预定义的模板进行自动替换和填充，省下人工核对格式的工作

## 说明

> (以`键：值 / key: value`代指本项目的替换对象)

目前翻译模式是以`data/`文件夹下的这几个匹配文件控制的：
- `keys.map`：键
- `effects.map`：食品/饮品的效果
- `manufacturers.map`：制造商
- `num_templates.map`：数字模板（数字使用`[NUM]`代替）
- [ ] (TODO) `values.map`：一般值的匹配

另外，需要按照`example_config.yaml`的格式在根目录下创建一个`config.yaml`保存部分设置

## 处理规则

**通用处理**：
- 对应如果有重复的，目前一般以已有的最多的一个为准
- 冒号多于一个的判断为可能是描述文本，跳过
- 对没有任何匹配的文本不进行填充，留空
- 填充后Paratranz保留未翻译状态

**对于键**：
- 目前主要以手动筛选为主

**对于值**：
- 只处理键在白名单里的（以防有些描述文本里也带有引号或者其他奇怪的edge cases）
- 厂商名统一为：`中文名+空格+英文括号(英文名)`
- 明确为各种数值的，使用模板进行翻译
- 食物/饮品的食用效果进行直接替换
- 其他暂时保留为原文
